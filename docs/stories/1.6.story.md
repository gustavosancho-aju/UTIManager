# Story 1.6: Fluxo de Audio e Extracao IA

**Status:** Done
**Sprint:** 1
**Assignee:** @dev
**Points:** 8

## Descricao
Como profissional de saude, quero gravar audio ou digitar texto com informacoes do paciente e ter os dados extraidos automaticamente pela IA, para que eu possa registrar evolucoes de forma rapida e hands-free.

## Criterios de Aceitacao
- [x] Pagina inicial /audio com selecao de paciente (lista pesquisavel)
- [x] Pagina de gravacao /audio/gravar com modo dual: Audio (microfone) ou Texto (entrada manual)
- [x] Reconhecimento de fala com transcricao ao vivo em portugues (pt-BR)
- [x] Envio da transcricao para API Gemini e armazenamento em sessionStorage
- [x] Pagina de revisao /audio/revisar com campos editaveis extraidos (7 secoes)
- [x] Criacao de paciente (se novo) e relatorio no Supabase ao confirmar
- [x] Fluxo completo funcional: selecao -> gravacao -> revisao -> salvamento

## Arquivos Modificados
- `src/app/(dashboard)/audio/page.tsx`
- `src/app/(dashboard)/audio/gravar/page.tsx`
- `src/app/(dashboard)/audio/revisar/page.tsx`
- `src/lib/supabase/reports.ts`

## Notas Tecnicas
Fluxo usa sessionStorage para passar dados entre paginas sem perder estado. Hook useSpeechRecognition gerencia o reconhecimento de fala. Dados extraidos pelo Gemini sao apresentados em formulario editavel antes de salvar. Suporte a criacao de novo paciente inline quando nao existe no sistema.
