# Story 1.4: API Proxy IA (Gemini)

**Status:** Done
**Sprint:** 1
**Assignee:** @dev
**Points:** 3

## Descricao
Como desenvolvedor, quero ter uma rota server-side que proteja a API key do Gemini e extraia dados medicos estruturados de transcricoes, para que o frontend nao exponha credenciais e receba dados parseados.

## Criterios de Aceitacao
- [x] Rota /api/ai/extract criada como server-side Route Handler
- [x] API key do Gemini protegida no servidor (nunca exposta ao client)
- [x] Transcricao enviada ao Gemini 3.1 Flash Lite Preview
- [x] System prompt otimizado para extracao de dados medicos em JSON estruturado
- [x] Retorno com dados extraidos: genero, leito, sinais vitais, dispositivos, ventilacao, sedacao
- [x] Tratamento de erros com respostas HTTP adequadas
- [x] Vercel configurado com regiao gru1 e headers de seguranca

## Arquivos Modificados
- `src/app/api/ai/extract/route.ts`
- `vercel.json`

## Notas Tecnicas
System prompt instrui o modelo a retornar JSON com campos especificos para dados de UTI. A rota valida o body da requisicao, envia ao Gemini via REST API e parseia a resposta. Vercel configurado para deploy na regiao gru1 (Sao Paulo) para menor latencia.
