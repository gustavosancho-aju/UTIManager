# Story 1.5: CRUD de Pacientes

**Status:** Done
**Sprint:** 1
**Assignee:** @dev
**Points:** 8

## Descricao
Como profissional de saude, quero gerenciar pacientes com listagem, cadastro, visualizacao, edicao e exclusao, para que eu possa manter o registro atualizado dos pacientes internados na UTI.

## Criterios de Aceitacao
- [x] Funcoes Supabase implementadas: getPatients, getPatient, createPatient, updatePatient, deletePatient, getPatientWithReports
- [x] Pagina de listagem /pacientes com tabela, GenderBadge e busca
- [x] Formulario de novo paciente /pacientes/novo com grid de 2 colunas
- [x] Pagina de detalhes /pacientes/[id] com identificacao, dados clinicos e tabela de relatorios
- [x] Pagina de edicao /pacientes/[id]/editar com formulario pre-preenchido
- [x] Exclusao com dialogo de confirmacao
- [x] Navegacao entre paginas funcional

## Arquivos Modificados
- `src/lib/supabase/patients.ts`
- `src/app/(dashboard)/pacientes/page.tsx`
- `src/app/(dashboard)/pacientes/novo/page.tsx`
- `src/app/(dashboard)/pacientes/[id]/page.tsx`
- `src/app/(dashboard)/pacientes/[id]/editar/page.tsx`

## Notas Tecnicas
Funcoes de acesso ao banco usam o client server do Supabase. Formularios utilizam Server Actions para submit. Busca implementada com query parameter e filtro server-side. Exclusao usa soft confirmation via Dialog do shadcn/ui.
